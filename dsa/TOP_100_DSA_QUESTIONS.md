## Problem-Solving Strategies

### For 10+ Years Experience Engineers

1. **Pattern Recognition**
   - Identify common patterns (sliding window, two pointers, etc.)
   - Map problems to known algorithms
   - Recognize when to use DP vs Greedy

2. **Optimization Focus**
   - Start with brute force, then optimize
   - Consider time-space tradeoffs
   - Think about edge cases and constraints

3. **Code Quality**
   - Write clean, readable code
   - Use meaningful variable names
   - Add comments for complex logic
   - Consider maintainability

4. **System Design Integration**
   - Think about scalability
   - Consider distributed systems implications
   - Discuss tradeoffs and alternatives

5. **Communication**
   - Explain your thought process
   - Discuss multiple approaches
   - Ask clarifying questions
   - Walk through examples

## Interview Tips

1. **Clarify Requirements**
   - Ask about input constraints
   - Understand edge cases
   - Confirm expected output format

2. **Think Aloud**
   - Explain your approach
   - Discuss tradeoffs
   - Show problem-solving process

3. **Start Simple**
   - Begin with brute force
   - Optimize incrementally
   - Discuss complexity at each step

4. **Test Your Solution**
   - Walk through examples
   - Consider edge cases
   - Verify correctness

5. **Handle Follow-ups**
   - Be ready for optimizations
   - Consider variations
   - Discuss real-world applications
